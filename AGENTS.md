# AGENTS.md — エージェント向け開発方針

このリポジトリでは、すべてのエージェント（Codex を含む）は以下の開発方針に従うこと。

## 1. 開発方針
- シンプルで読みやすい構成を最優先する。
- 複雑化を避け、可能な限り小さく明確な関数・クラスで構築する。
- 文脈を推測しにくいコードは避け、明確な意図が読み取れる実装とする。

## 2. コーディング規約
- Python では型ヒントを必須とする。
- 例外処理は明示的に書き、曖昧な `except:` は禁止。
- 変数名・関数名は役割が明確に分かる名称にする。
- コメントは「なぜこの実装なのか」を中心に丁寧に残す。
- 自動生成コードであっても、読めるコードを心がける。

## 3. テスト
- 単体テスト（pytest）を含めることを必須とする。
- ロジック部分はできる限りテスト可能な構造に分離する。
- 外部 API 呼び出しはモック化してテストする。

## 4. レビュー（Codex への指示）
- 可読性の低下につながる変更がないかを最優先で確認する。
- シンプルな構造から逸脱していないかを確認する。
- コメントが不足している場合は追加を促す。
- 仕様に関わる判断は README.md を参照すること。

以上。
エージェントはこのファイルのみを開発ルールの基準とする。
